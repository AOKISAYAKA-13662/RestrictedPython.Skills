# PM（プロジェクトマネージャー）スキル

## 役割
ERPからKyribaへのファイル変換Restricted Pythonスクリプト開発プロジェクトの全体統括。
3つの担当スキルに指示を出し、成果物をつなぎ、プロジェクトを完了まで導く。

## プロジェクト全体フロー

```
Phase 1: 要件定義・設計（business-analystスキル）
    ↓ 成果物: 変換仕様書
Phase 2: スクリプト開発（developerスキル）
    ↓ 成果物: Restricted Pythonスクリプト
Phase 3: レビュー（business-analystスキル で仕様適合確認）
    ↓ 成果物: レビュー結果・修正依頼
Phase 4: テスト（testerスキル）
    ↓ 成果物: テストシナリオ・テスト結果報告
Phase 5: 修正・再テスト（必要に応じてPhase 2-4を繰り返す）
Phase 6: リリース判定
```

## PMの具体的な作業

### 1. プロジェクト開始時
ユーザーから以下の情報をヒアリングする:
- ERPの種類（SAP, Oracle, etc.）
- 連携するファイルの種類と形式（CSV, TXT, XML等）
- Kyribaで取り込む先の機能（支払い、キャッシュポジション、銀行残高等）
- 変換の概要（何をどう変えたいか）
- スケジュール・納期

### 2. 各フェーズでの指示出し

#### Phase 1 を開始するとき
business-analystスキルに以下を伝えて実行させる:
```
business-analystスキルを使って、以下の変換仕様書を作成してください。
- 入力ファイル: [ERPからのファイル情報]
- 出力ファイル: [Kyribaが求める形式]
- 変換要件: [ユーザーからヒアリングした内容]
```

#### Phase 2 を開始するとき
developerスキルに変換仕様書を渡して実行させる:
```
developerスキルを使って、以下の変換仕様書に基づくRestricted Pythonスクリプトを作成してください。
- 変換仕様書: [Phase 1の成果物]
```

#### Phase 3 を開始するとき
business-analystスキルにレビューを依頼する:
```
business-analystスキルを使って、以下のスクリプトが変換仕様書に適合しているかレビューしてください。
- 変換仕様書: [Phase 1の成果物]
- スクリプト: [Phase 2の成果物]
```

#### Phase 4 を開始するとき
testerスキルに以下を伝えて実行させる:
```
testerスキルを使って、以下のスクリプトのテストシナリオを作成し、テストを実施してください。
- 変換仕様書: [Phase 1の成果物]
- スクリプト: [Phase 2の成果物]
```

### 3. 進捗管理
各フェーズの完了時に以下を確認する:
- 成果物が揃っているか
- 次のフェーズに進める状態か
- ユーザーの承認が必要か

### 4. 問題発生時
- テストでNGが出た場合 → developerスキルに修正を依頼
- 仕様の曖昧さが見つかった場合 → business-analystスキルに仕様の明確化を依頼
- 想定外のデータパターンが見つかった場合 → ユーザーに確認

## 成果物管理

各フェーズの成果物は以下のディレクトリに保存する:
```
project/
├── 01_design/
│   ├── conversion_spec.md        # 変換仕様書
│   └── review_result.md          # レビュー結果
├── 02_development/
│   └── transform_script.py       # Restricted Pythonスクリプト
├── 03_test/
│   ├── test_scenarios.md         # テストシナリオ
│   ├── test_data/                # テストデータ
│   │   ├── input_normal.csv      # 正常系テストデータ
│   │   ├── input_error.csv       # 異常系テストデータ
│   │   └── expected_output.csv   # 期待値データ
│   └── test_report.md            # テスト結果報告
└── 04_release/
    └── release_note.md           # リリースノート
```

## リリース判定基準
以下がすべて満たされたらリリース可とする:
- 全テストシナリオがPASS
- 業務担当者のレビュー承認済み
- ユーザーの最終承認済み

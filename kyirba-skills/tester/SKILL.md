# テスト担当者（Tester）スキル

## 役割
変換仕様書とスクリプトを受け取り、テストシナリオの作成・テストデータの準備・テスト実施・結果報告を行う。

## 作業の流れ

```
1. 変換仕様書を読み、テスト観点を洗い出す
2. テストシナリオを作成する
3. テストデータ（入力＋期待値）を作成する
4. スクリプトを実行してテストする
5. 結果を判定し、報告書を作成する
```

## 作業1: テストシナリオの作成

### テスト観点

以下の3カテゴリでテストケースを作る:

#### A. 正常系テスト
変換仕様書通りのデータが正しく変換されることを確認する。

考慮するポイント:
- 全項目が正しい値で埋まっているデータ
- 各変換ルールが個別に正しく動くか
- 複数の変換ルールが組み合わさった場合に正しく動くか
- 複数行のデータが正しく処理されるか
- フィルタリングルールで対象行だけが出力されるか

#### B. 異常系テスト
想定外のデータが来たとき、仕様書の例外処理ルール通りに動作することを確認する。

考慮するポイント:
- 必須項目が空のケース
- 数値項目に文字が入っているケース
- 日付項目に不正な値が入っているケース（13月、32日等）
- 想定外のコード値が入っているケース
- ファイルが空（0行）のケース
- ヘッダーのみで明細がないケース
- 極端に長い文字列が入っているケース

#### C. 境界値テスト
ギリギリのケースで正しく動作するか確認する。

考慮するポイント:
- 金額が0のケース
- 金額がマイナスのケース
- 日付が月末（1/31, 2/28, 2/29, 12/31等）のケース
- 桁数が上限ぴったりのケース
- 1行だけのファイルのケース
- 大量行（性能面の確認）のケース

## 作業2: テストデータの作成

テストシナリオに基づき、以下のファイルを作成する:

### 入力テストデータ
- `03_test/test_data/input_normal.csv` — 正常系テスト用
- `03_test/test_data/input_error.csv` — 異常系テスト用
- `03_test/test_data/input_boundary.csv` — 境界値テスト用

### 期待値データ
- `03_test/test_data/expected_normal.csv` — 正常系の期待出力
- `03_test/test_data/expected_error.csv` — 異常系の期待出力
- `03_test/test_data/expected_boundary.csv` — 境界値の期待出力

テストデータ作成のルール:
- 変換仕様書のサンプルデータを基にする
- 各テストケースが独立して検証できるようにする
- 1ファイルに複数テストケースをまとめてもよいが、どの行がどのケースかコメントする

## 作業3: テスト実施

### テスト手順

1. スクリプトと入力テストデータを用意する
2. Kyriba環境（またはローカルのPython環境）でスクリプトを実行する
3. 出力結果と期待値データを比較する
4. 一致すればPASS、不一致ならFAILとして記録する

### ローカルテスト用のヘルパー（Kyriba外で事前検証する場合）

```python
# test_runner.py（通常のPythonで実行するテスト補助スクリプト）
import csv

def load_csv(filepath):
    """CSVファイルをリストとして読み込む"""
    with open(filepath, 'r', encoding='utf-8') as f:
        reader = csv.reader(f)
        return [row for row in reader]

def compare_output(actual, expected):
    """実際の出力と期待値を比較する"""
    if len(actual) != len(expected):
        return False, f"行数不一致: 実際={len(actual)}, 期待={len(expected)}"
    
    for i, (act_row, exp_row) in enumerate(zip(actual, expected)):
        if act_row != exp_row:
            return False, f"行{i}不一致: 実際={act_row}, 期待={exp_row}"
    
    return True, "一致"

# テスト実行
input_data = load_csv("test_data/input_normal.csv")
expected = load_csv("test_data/expected_normal.csv")
```

## テスト完了条件
- 全テストケースが実施済み
- FAIL項目がゼロ（または許容範囲内で業務担当者の承認あり）
- テスト結果報告書が完成している
